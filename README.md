# Digital-Check-In-System-via-Face-Recognition
My project works to implement an automated face recognition-based attendance system to overcome these challenges. This system tracks attendance via human congestion minimization and facial recognition technology. This offers a quick, secure, and speedy means of marking attendance that helps improve operational efficiency and eradicate fraudulent practices.

## Technologies Used (Django Framework)
1. HTML, CSS, JavaScript, and Bootstrap for front-end.
2. Python for backend
3. MySQL for the database
4. OpenCV and face-recognition for face recognition

## Features
### a) User Management
1.	User registration: register new users with personal information and facial images.
2.	Facial Recognition: Authenticate users using facial recognition as part of check-in user verification.
3.	Profile Management allows users to view and update their profiles.
4.	RBAC (Role-Based Access Control): Allows the use of different user roles (admin, employee) to give access to view and edit the attendance data.
### b) Face Recognition
5.	Webcam Input: Detecting faces in webcam input (real-time).
6.	Verification: Captured face and stored facial data match for verification.
### c) Attendance Management
7.	Automated Check-In and Check-Out: Identify faces and automatically register attendance, allowing the teachers to be free to attend their classes.
8.	Attendance History: This will display the attendance history of individual employees.
9.	Attendance Override by Admins: This allows admins to adjust attendance should they need it.
10.	Leave
11.	Apology
### d) Reporting and Analytics
12.	Attendance Reports: Generate detailed attendance reports by date, department, or employee.
13.	Dashboard Analytics: Provide a dashboard with analytics on attendance trends and patterns.
14.	Attendance Exports: Attendance reports can be exported in formats like CSV.
### e) Accessibility
 13. Translated to multiple languages
14. Mobile Responsiveness
